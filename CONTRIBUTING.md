# Contributing

Thanks for helping improve FS42 Scheduler.

## Keep The Project Lightweight

- Keep changes focused on the existing static app.
- Avoid adding frameworks, package managers, or build tooling unless there is a clear need.
- Preserve the single-user planning workflow.

## Before You Open A Change

- Test the app in a browser.
- Check timeline drag/drop, snapping, resize, validation, and export paths.
- Keep export wording and FS42-facing behaviour consistent with the README.

## Good Change Shape

- Small fixes to scheduling, validation, usability, or export readiness.
- Documentation updates that help another FS42 user understand the tool quickly.
- Repo hygiene improvements that keep the project easy to share.
